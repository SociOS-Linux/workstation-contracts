# M2 IPC Protocol v1.0 (Runner ↔ Adapter)

Status: Draft v1.0  
Transport: JSON-over-stdio (newline-delimited JSON / NDJSON)  
Scope: `contract-runner` ↔ backend adapter subprocess contract only

## Purpose

This protocol defines the stable execution seam between the workstation contract runner
and backend adapters. It exists so adapters can be implemented in any OSS language
without changing runner semantics.

This protocol is for:
- adapter discovery and capability checks
- lock validation and hashing
- environment realization
- task execution
- dependency inventory for licensing / evidence gates

This protocol is **not**:
- the workspace controller
- a remote/network RPC protocol
- the image build substrate

## Transport and framing

- runner spawns adapter executable, typically `adapters/<backend>/adapter`
- requests are written as NDJSON to adapter `stdin`
- responses are written as NDJSON to adapter `stdout`
- human logs go to `stderr` only
- one JSON object per line

## Mandatory handshake

Runner MUST send `hello` immediately after spawn.
Adapter MUST respond before any other operation is accepted.

If an operation is attempted before a successful `hello`, the adapter MUST fail with
`E_HELLO_REQUIRED`.

## Versioning

`protocol` uses `MAJOR.MINOR`.

- MAJOR = breaking change
- MINOR = backward-compatible additive change

Rules:
- receivers MUST ignore unknown fields
- removing or renaming fields requires a MAJOR bump
- adding optional fields or capabilities is a MINOR bump

v1.0 is the first stable wire version.

## Common envelopes

### Request envelope

```json
{
  "id": "uuid-or-monotonic",
  "op": "hello|caps|info|lock_validate|lock_hash|env_realize|task_run|deps_inventory|lock_update|env_shell|ping",
  "protocol": "1.0",
  "ts": "2026-04-20T00:00:00Z",
  "lane": {"lane_id": "linux-amd64-container"},
  "role": "ci",
  "task": "build",
  "ci_mode": true,
  "params": {}
}
```

### Response envelope

```json
{
  "id": "same-as-request",
  "op": "same-as-request-op-or-error",
  "protocol": "1.0",
  "ts": "2026-04-20T00:00:00Z",
  "ok": true,
  "result": {},
  "errors": [],
  "warnings": []
}
```

Rules:
- `id` and `op` are always required
- if `ok` is `false`, `errors` MUST be non-empty
- paths returned in results are repo-relative
- receivers MUST ignore unknown fields

## LaneConfig

Runner passes a fully resolved lane when the operation needs it.

```json
{
  "lane_id": "linux-amd64-container",
  "platform": "linux",
  "arch": "amd64",
  "backend": "pixi",
  "lock_ref": "workstation.lock/pixi.linux-amd64.lock",
  "containerized": true,
  "container": {
    "image": "debian@sha256:...",
    "runtime": "podman",
    "mount_repo_to": "/repo",
    "workdir": "/repo"
  }
}
```

All paths are interpreted relative to repo root.

## Operations

### `hello`
Negotiates protocol compatibility and required capabilities.

Runner request carries:
- runner identity
- supported/desired protocol version
- required capabilities
- optional capabilities

Adapter response carries:
- adapter identity
- supported protocol range
- selected version
- supported capabilities
- missing required capabilities

If no overlap exists, fail with `E_PROTOCOL_INCOMPATIBLE`.

### `caps`
Returns capability versions/notes.

### `info`
Returns adapter identity and config pattern.

### `lock_validate`
Returns pass/fail plus notes.

### `lock_hash`
Returns canonical lane `sha256`.

### `env_realize`
Ensures the environment for the given role is available.

### `task_run`
Executes the backend-native task and returns:
- exit code
- duration
- stdout/stderr log paths

### `deps_inventory`
Produces dependency inventory for license/evidence gates and returns the written path.

## Error object

```json
{
  "code": "E_*",
  "message": "human readable",
  "details": {}
}
```

Canonical error families in v1.0:
- `E_PROTOCOL_INCOMPATIBLE`
- `E_HELLO_REQUIRED`
- `E_PROTOCOL_MISMATCH`
- `E_CAPABILITY_MISSING`
- `E_LOCK_MISSING`
- `E_LOCK_INVALID`
- `E_LOCK_HASH_FAILED`
- `E_ENV_REALIZE_FAILED`
- `E_ENV_FLOATING_DISALLOWED`
- `E_TASK_UNKNOWN`
- `E_TASK_FAILED`
- `E_TASK_TIMEOUT`
- `E_CMD_FAILED`
- `E_PATH_ESCAPE`
- `E_WRITE_DENIED`
- `E_DEPS_INVENTORY_FAILED`
- `E_UNSUPPORTED`
- `E_INTERNAL`

## Fixtures

The fixture directory is the executable compatibility surface for the protocol:

- `fixtures/m2-ipc/v1.0/hello_ok.ndjson`
- `fixtures/m2-ipc/v1.0/hello_incompatible.ndjson`
- `fixtures/m2-ipc/v1.0/capability_missing.ndjson`
- `fixtures/m2-ipc/v1.0/lock_hash_ok.ndjson`
- `fixtures/m2-ipc/v1.0/task_run_ok.ndjson`
- `fixtures/m2-ipc/v1.0/task_run_fail.ndjson`
- `fixtures/m2-ipc/v1.0/deps_inventory_ok.ndjson`

These are contract vectors for both runner and adapter implementations.

## Deferred to next tranche

- per-operation schemas beyond the common envelope
- a fixture replay harness
- remote/network transport
- adapter sandboxing/authentication

Those should follow once the wire contract is accepted.
